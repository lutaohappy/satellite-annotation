# 部署问题记录

## 2026-04-15: 静态资源配置问题

### 问题现象
- 浏览器访问 `http://192.168.200.77:4000/gis/` 显示旧版本页面
- 控制台报错 `watch is not defined`
- 浏览器加载的文件名与服务器上不一致（如 `MapView-B4JgIolL.js`）

### 根本原因
1. **WebConfig 中静态资源路径配置错误**
   - 开发环境路径：`/Users/taolu/Downloads/test/satellite-annotation/frontend/dist/`
   - 生产环境路径：`/opt/satellite-annotation/frontend/`
   - 配置文件使用了开发路径，导致服务器上无法正确映射静态资源

2. **浏览器缓存问题**
   - Vite 构建的文件名带有 hash（如 `MapView-Bq-IqMct.js`）
   - 浏览器缓存了旧版本的文件
   - 即使服务器更新了，浏览器仍加载缓存的旧文件

### 解决方案

#### 1. 修改 WebConfig.java
```java
// 错误配置（开发环境路径）
private static final String FRONTEND_DIST_PATH = "/Users/taolu/Downloads/test/satellite-annotation/frontend/dist/";

// 正确配置（生产环境路径）
private static final String FRONTEND_DIST_PATH = "/opt/satellite-annotation/frontend/";
```

**注意**：路径格式应为 `/opt/satellite-annotation/frontend/`，不要加 `file:` 前缀
（Spring 的 `addResourceLocations()` 方法会自动添加 `file:` 前缀）

#### 2. 重新构建并部署
```bash
# 构建前端
cd /Users/taolu/Downloads/test/satellite-annotation/frontend
npm run build

# 复制前端文件到服务器
sshpass -p '密码' scp -r dist/* root@192.168.200.77:/opt/satellite-annotation/frontend/

# 构建后端
cd /Users/taolu/Downloads/test/satellite-annotation/backend
mvn clean package -DskipTests

# 复制后端 jar 到服务器
sshpass -p '密码' scp target/satellite-annotation-backend-0.0.1-SNAPSHOT.jar root@192.168.200.77:/opt/satellite-annotation/

# 重启后端
sshpass -p '密码' ssh root@192.168.200.77 "cd /opt/satellite-annotation && pkill -f satellite-annotation-backend && sleep 2 && nohup java -jar satellite-annotation-backend-0.0.1-SNAPSHOT.jar > backend.log 2>&1 &"
```

#### 3. 清除浏览器缓存
用户必须执行以下操作之一：

**方法 A：使用开发者工具**
1. 按 `F12` 打开开发者工具
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

**方法 B：清除站点数据**
1. 访问 `chrome://settings/siteData?searchSubpage=192.168.200.77`
2. 点击"删除"清除该站点的所有缓存数据

**方法 C：快捷键**
- Windows/Linux: `Ctrl + Shift + Delete`
- Mac: `Cmd + Shift + Delete`

#### 2. 检查页面版本
页面右下角应显示版本号，如：
```
版本：20260415-31-路网异步下载
```

#### 3. 检查控制台日志
按 `F12` 打开控制台，查看启动日志中的版本信息：
```
[MapView] 当前版本：20260415-31-路网异步下载
```

### 预防措施

1. **在 index.html 中添加版本号**
   - 每次部署时更新版本号
   - 便于快速识别是否为最新版本

2. **添加缓存控制头**
   ```java
   // WebConfig.java 中已添加 NoCacheFilter
   httpResponse.setHeader("Cache-Control", "no-cache, no-store, must-revalidate");
   httpResponse.setHeader("Pragma", "no-cache");
   httpResponse.setHeader("Expires", "0");
   ```

3. **使用版本号查询参数**
   - 在引用静态资源时添加版本参数
   - 如：`<script src="/src/main.js?v=20260415-31">`

4. **建立标准部署流程**
   - 前端构建 → 复制 dist 到服务器
   - 后端构建 → 复制 jar 到服务器
   - 重启服务 → 验证版本
   - 通知用户 → 清除缓存

### 部署脚本（建议）

```bash
#!/bin/bash
# deploy.sh - 自动化部署脚本

SERVER="192.168.200.77"
USER="root"
PASSWORD="密码"

echo "=== 构建前端 ==="

npm run build

cd ../backend
mvn clean package -DskipTests

echo "=== 部署后端 ==="
sshpass -p "$PASSWORD" scp target/satellite-annotation-backend-0.0.1-SNAPSHOT.jar $USER@$SERVER:/opt/satellite-annotation/
```
